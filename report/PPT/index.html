性能指标与设计约束

— PWM 方波

1 Hz–100 kHz · 50% 占空比

上限取 100 kHz：兼顾测频精度与电网频率测量需求

— DAC 正弦波

1–1000 Hz · Amp 0–2047 · Phase 0–359°

12 位 DAC 码值 0–4095，以 2048 对应 1.65 V 偏置中点

— ADC 采样

三通道 · 128 点 · 6400 Hz

按 50 Hz 市电设计：50 × 128 = 6400 Hz

— 采样通道

电压 · 电流 · 漏电电流

三路信号经 miniIO 调理后进入 STM32 ADC 通道

— 串口通信

115200 bps · REPORT 约 1 s

限制主动上报频率，避免串口输出影响主循环调度

— LCD 显示

两种触发策略

固定刷新频率 3 Hz / 事件驱动触发

— 计算约束

Cortex-M3 无 FPU · 定点/整型算法

避免 FFT/DFT 浮点运算开销，直接计算 RMS、P、S、PF

— 存储约束

静态缓冲 · 固定帧长 · 无动态内存

所有采样帧、通信缓冲和状态结构体编译期分配

04 / 通信协议设计与验证 · 设计

二进制帧 + ASCII Payload

串口通信基础设施 · 帧边界识别 · 可变长命令 · CRC 校验
字段 长度 说明
SOF 2B 固定 0xA5 0x5A，互为反码
VERSION 1B 协议版本，当前为 0x01
TYPE 1B 01 命令 / 81 响应 / 82 事件 / 83 错误
SEQ 1B 帧序号，请求与响应对应
LEN 2B Payload 长度，小端，最大 512B
PAYLOAD LEN 可打印 ASCII 文本命令或状态数据
CRC16 2B CRC-16/Modbus，小端

CRC 覆盖 VERSION → PAYLOAD，不覆盖 SOF 和 CRC 本身；启动后发送 TYPE=0x82, PAYLOAD="OK DEVICE READY"。

为什么这样设计

A5 5A 帧头：1010/0101 交替，便于识别帧边界
LEN 字段：支持可变长 payload，后续命令易扩展
CRC-16/Modbus：检测误码，避免错误命令被执行
ASCII payload：保留文本命令的可读性，方便串口调试

COMMAND SET
app_command.c
帧内承载的 ASCII 命令 
── 查询类 ──
HELP             → 列出支持的命令
STATUS?          → 查询全量状态快照
REPORT?          → 查询自动上报开关

TIM1 频率生成

设计任务

PA8 输出 1 Hz – 100 kHz 可调 50% 占空比方波，供测频和外部使用

外设

TIM1_CH1 (PA8) · APB2 72 MHz 时钟

$$ \begin{aligned} N &= 72\text{M} / f_{\text{target}} \\ \text{PSC} &= \lceil N / 65536 \rceil - 1 \\ \text{ARR} &= 72\text{M} / (f \times (\text{PSC}{+}1)) - 1 \\ \text{CCR} &= (\text{ARR} + 1) / 2 \end{aligned} $$
PWM CONFIG
app_pwm.c
频率设置核心逻辑 
void app_pwm_set_freq(uint32_t freq) {
    uint32_t n  = 72000000UL / freq;
    uint16_t psc = (n - 1) / 65536;
    uint32_t arr = 72000000UL /
        (freq * (psc + 1)) - 1;
    TIM1->PSC  = psc;
    TIM1->ARR  = arr;
    TIM1->CCR1 = (arr + 1) / 2;
}
调试要点

KEY0/KEY1 按键调整频率 → LCD 实时显示 → PA8→PA1 环回自测验证

05 / 频率生成与测量

TIM2 频率测量

设计任务

PA1/TIM2_CH2 输入捕获，测量 1 Hz – 100 kHz 方波频率；LCD 显示分辨率 0.01 Hz

测频原理

TIM2 以 1 MHz 计数，捕获相邻上升沿计数差 $T(\mu s)$，频率 $f {\times} 100 = 10^8 / T$

溢出扩展

16 位计数器溢出累加 s_overflows；捕获瞬间若 Update 标志置位且 CCR2 < 32768，高位自动 +1

超时归零

2 秒无捕获自动清零，避免 LCD 和串口继续保留旧频率。

make_capture_time() 时间戳拼接 
static uint32_t make_capture_time(uint16_t ccr)
{
    uint16_t hi = s_overflows;
    // 溢出已发生但 ISR 尚未执行
    if ((TIM2->SR & TIM_FLAG_Update)
        && ccr < 32768) {
        hi++;
    }
    return ((uint32_t)hi << 16) | ccr;
}

period = now_ts - last_ts;  // μs
freq_x100 = 100000000UL / period;

128 点查找表 · DMA 循环 · 双通道独立

— 设计任务

双路正弦测试信号

PA4/PA5 输出 0–3.3 V 正弦波；不接入强电时，用 DAC 环回验证 ADC 采样、波形显示与电参量计算链路。

— 查找表

全幅正弦查找表

$$S[n] = 2048 + 2047 \cdot \sin\!\left(\frac{2\pi n}{128}\right)$$

编译期预计算 128 点，范围 0–4095，中心 2048（1.65V 偏置）

— 幅值/相位

实时缩放

$$S'[n] = 2048 + (S[n]-2048) \times \frac{A}{2047}$$

相位通过偏移索引实现：$\text{idx} = (n + \varphi {\times} 128 / 360) \bmod 128$

— DMA 传输

TIM6/7 触发 DMA

TIM6/TIM7 分别触发 CH1/CH2，定时器触发 DMA 循环输出，低 CPU 占用；两路频率可独立控制

06 / DAC 测试源与 ADC 采样 · 信号采集与 DMA

三通道准同步扫描与 DMA

实际信号来源（miniIO 副板）

VL → 电压互感器隔离 + 1.65 V 偏置调理 → PC0 / ADC_IN10
iL → 电流互感器采样 + 1.65 V 偏置调理 → PC3 / ADC_IN13
iLK → 漏电电流检测 + 1.65 V 偏置调理 → PC2 / ADC_IN12
fr → 市电整形方波 → PA1 / TIM2_CH2 输入捕获

采样目标

面向 50 Hz 工频信号，每通道 128 点/周期，用于 RMS、有功功率和功率因数计算。

触发与采样率

TIM3 TRGO 以 6400 Hz 触发 ADC1；每次触发完成 3 通道序列转换，所以每通道采样率为 6400 Hz。

通道扫描顺序

Rank1: PC0/VL → Rank2: PC3/iL → Rank3: PC2/iLK；单通道转换约 20 μs，三通道序列约 60 μs，相对 50 Hz 周期可视为准同步。

DMA 传输

DMA1_CH1 循环模式，缓冲区 768 个 halfword = 2 帧 × 128 点 × 3 通道；HT 处理中前半帧，TC 处理中后半帧。

ADC ISR
app_adc.c
ISR deinterleave + ping-pong 帧缓存 
static void deinterleave_frame(
    const uint16_t *src)
{
    uint8_t wi = s_adc_write_idx;

    for (uint16_t k = 0; k < 128; k++) {
        s_adc_frame[wi].v[k]   = src[k * 3 + 0];
        s_adc_frame[wi].i[k]   = src[k * 3 + 1];
        s_adc_frame[wi].ilk[k] = src[k * 3 + 2];
    }

    s_adc_ready_idx = wi;
    s_adc_write_idx = wi ^ 1U;
    s_adc_ready     = 1U;
}
关键设计

DMA circular 不切换 CMAR；HT/TC 中断只做固定长度解交错和 ready 标志切换，RMS、功率计算、LCD 刷新和串口发送都放到主循环。

07 / 电参量计算与显示

整型 RMS 与物理量换算

1. 直流分量（零偏）
$$Z = Z_{\text{calibrated}} \quad \text{(上电自动求均值校准，消除硬件偏差)}$$
2. 有效值码值（u64 累加防溢出）
$$\text{RMS} = \sqrt{\frac{\sum (x_i - Z)^2}{N}}$$
3. 物理量换算
$$V \times 100 = \frac{\text{RMS} \times G_v}{1000} \quad (G_v = 17600)$$

单位：vrms_x100 → 0.01V；irms_x1000 → 0.001A

POWER CALC
integer calc
有功/视在功率与功率因数
4. 有功功率
$$P = \frac{\sum (v_i - Z_v)(i_i - Z_i) \times G_v \times G_i}{10^{10} \times N}$$

i32 保留符号；P 保留符号，用于区分同相消耗与反相回馈

5. 视在功率 & 功率因数
$$S = V_{\text{rms}} \times I_{\text{rms}} \qquad \text{PF} = \frac{|P|}{S} \times 1000$$

S=0 时 PF 返回 0；PF 钳位 [0,1000]，避免数值误差导致超过物理范围

08 / 系统联调 · 问题排查

关键问题排查与工程修正

— 问题 01 · 硬件干扰

PC1 触摸屏引脚干扰

现象：电流通道接 PC1 时，阻抗异常导致电流采样值完全失真。
成因：开发板 PC1 连接了触摸屏芯片且存在硬上拉电阻干扰。
解决：重新分配引脚，将电流采样迁移至闲置的 PC3 通用通道。

— 问题 02 · 并发竞争

DMA 与 LCD 串口竞争

现象：高频 LCD 刷新和串口输出时，采样波形断裂、电参量数据偶发跳变。
成因：DMA 中断与主循环并发访问同一缓冲区。
解决：设计“双缓冲+工作帧+显示快照”三重缓冲机制实现读写隔离。

— 问题 03 · 算法极性

CT 极性相反导致 P 符号异常

现象：正常阻性负载下，有功功率 P 反向计算为负值，PF 正常。
成因：电流互感器(CT)接线极性反接，导致电流采样信号反相 180°。
解决：在计算库中增加极性配置宏，使用 APP_ADC_I_POLARITY = -1 软件修正。

— 问题 04 · 初始化遗漏

继电器未正确初始化

现象：联机后副板继电器无法正常闭合，导致测量电参量均为 0。
成因：初始化逻辑 app_init() 中遗漏了 app_relay_init() 调用。
解决：在初始化入口中补全调用，确保上电后默认安全断开并合理吸合。
