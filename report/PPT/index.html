02 / PWM 方波发生

TIM1 频率生成

设计任务

PA8 输出 1 Hz – 100 kHz 可调 50% 占空比方波，供测频和外部使用

外设

TIM1_CH1 (PA8) · APB2 72 MHz 时钟

$$ \begin{aligned} N &= 72\text{M} / f_{\text{target}} \\ \text{PSC} &= \lceil N / 65536 \rceil - 1 \\ \text{ARR} &= 72\text{M} / (f \times (\text{PSC}{+}1)) - 1 \\ \text{CCR} &= (\text{ARR} + 1) / 2 \end{aligned} $$
PWM CONFIG
app_pwm.c
频率设置核心逻辑（3 行） 
void app_pwm_set_freq(uint32_t freq) {
    uint32_t n  = 72000000UL / freq;
    uint16_t psc = (n - 1) / 65536;
    uint32_t arr = 72000000UL /
        (freq * (psc + 1)) - 1;
    TIM1->PSC  = psc;
    TIM1->ARR  = arr;
    TIM1->CCR1 = (arr + 1) / 2;
}

03 / 输入捕获测频

高精度测频与 32 位时间戳扩展

设计任务

PA1/TIM2_CH2 输入捕获，测量 1 Hz – 100 kHz 方波频率，精度 0.01 Hz

测频原理

TIM2 以 1 MHz 计数，捕获相邻上升沿计数差 $T(\mu s)$，频率 $f {\times} 100 = 10^8 / T$

溢出扩展

16 位计数器溢出累加 s_overflows；捕获瞬间若 Update 标志置位且 CCR2 < 32768，高位自动 +1

超时与节流

2 秒无捕获自动清零；上报节流：每秒周期 + 变化即报（最小 200 ms 间隔）

make_capture_time() 时间戳拼接 

        static uint32_t make_capture_time(uint16_t ccr)
        {
            uint16_t hi = s_overflows;
            // 溢出已发生但 ISR 尚未执行
            if ((TIM2->SR & TIM_FLAG_Update)
                && ccr < 32768) {
                hi++;
            }
            return ((uint32_t)hi << 16) | ccr;
        }
        // 频率计算
        period = now_ts - last_ts;  // μs
        freq_x100 = 100000000UL / period;

04 / DAC 正弦波发生

128 点查找表 · DMA 循环 · 双通道独立

— 设计任务

双路正弦测试信号

PA4/PA5 输出 0–3.3V 正弦波，为 ADC 环回测试提供安全信号源；频率/幅度/相位独立可调

— 查找表

全幅正弦查找表

$$S[n] = 2048 + 2047 \cdot \sin\!\left(\frac{2\pi n}{128}\right)$$

编译期预计算 128 点，范围 0–4095，中心 2048（1.65V 偏置）

— 幅值/相位

实时缩放

$$S'[n] = 2048 + (S[n]-2048) \times \frac{A}{2047}$$

相位通过偏移索引实现：$\text{idx} = (n + \varphi {\times} 128 / 360) \bmod 128$

— DMA 传输

TIM6/7 触发 DMA

TIM6/TIM7 分别触发 CH1/CH2，驱动 DMA 双通道循环输出，零 CPU 占用；频率分别独立控制

05 / ADC 多通道采样

三通道扫描与 DMA

设计任务

同一触发源下准同步顺序扫描采样三路交流信号，每周期 128 点，供 RMS 和功率计算

触发源

TIM3 TRGO 以 6400 Hz 触发 ADC1（50 Hz × 128 点/周期）

通道扫描顺序

Rank1: PC0/IN10(VL) → Rank2: PC3/IN13(iL) → Rank3: PC2/IN12(iLK)；采样 239.5 cycles（~20 μs）

DMA 传输

DMA1_CH1 循环模式，缓冲 768 半字；HT 中断（前半帧）+ TC 中断（后半帧）

ADC ISR
app_adc.c
ISR deinterleave + 双缓冲切换 
static void deinterleave_frame(
    const uint16_t *src)
{
    uint8_t wi = s_adc_write_idx;
    for (uint16_t k = 0; k < 128; k++) {
        s_adc_frame[wi].v[k]   = src[k*3];
        s_adc_frame[wi].i[k]   = src[k*3+1];
        s_adc_frame[wi].ilk[k] = src[k*3+2];
    }
    s_adc_ready_idx = wi;
    s_adc_write_idx = wi ^ 1;
    s_adc_ready     = 1;
}

06 / 电参量计算

整型 RMS 与物理量换算

1. 直流分量（零偏）
$$Z = Z_{\text{calibrated}} \quad \text{(上电自动求均值校准，消除硬件偏差)}$$
2. 有效值码值（u64 累加防溢出）
$$\text{RMS} = \sqrt{\frac{\sum (x_i - Z)^2}{N}}$$
3. 物理量换算
$$V \times 100 = \frac{\text{RMS} \times G_v}{1000} \quad (G_v = 17600)$$

单位：vrms_x100 → 0.01V；irms_x1000 → 0.001A

* 注：DAC 环回测试采用单位增益验证算法趋势；miniIO 市电测量采用 Gv/Gi 校准增益。

POWER CALC
pm_calc.rs
有功/视在功率与功率因数
4. 有功功率
$$P = \frac{\sum (v_i - Z_v)(i_i - Z_i) \times G_v \times G_i}{10^{10} \times N}$$

i32 保留符号；除数 $10^{10}$ 来自 vrms(0.01V) × irms(0.001A) × gain²

5. 视在功率 & 功率因数
$$S = V_{\text{rms}} \times I_{\text{rms}} \qquad \text{PF} = \frac{|P|}{S} \times 1000$$

S=0 时 PF 返回 0（无负载时 PF 无意义）；PF 钳位 [0, 1000]

08 / 关键问题排查

调试中的四类关键技术问题与闭环排查

— 问题 01 · 硬件干扰

PC1 触摸屏引脚干扰

现象：电流通道接 PC1 时，阻抗异常导致电流采样值完全失真。
成因：开发板 PC1 连接了触摸屏芯片且存在硬上拉电阻干扰。
解决：重新分配引脚，将电流采样迁移至闲置的 PC3 通用通道。

— 问题 02 · 并发竞争

DMA 与 LCD 串口竞争

现象：高频 LCD 刷新和串口输出时，采样波形断裂、电参量数据偶发跳变。
成因：多线程前后台并发读写同一块 DMA 缓冲区。
解决：设计“双缓冲+工作帧+显示快照”三重缓冲机制实现读写隔离。

— 问题 03 · 算法极性

CT 极性相反导致 P 符号异常

现象：正常阻性负载下，有功功率 P 反向计算为负值，PF 正常。
成因：电流互感器(CT)接线极性反接，导致电流采样信号反相 180°。
解决：在计算库中增加极性配置宏，使用 APP_ADC_I_POLARITY = -1 软件修正。

— 问题 04 · 初始化遗漏

继电器未正确初始化

现象：联机后副板继电器无法正常闭合，导致测量电参量均为 0。
成因：初始化逻辑 app_init() 中遗漏了 app_relay_init() 调用。
解决：在初始化入口中补全调用，确保上电后默认安全断开并合理吸合。
