\vspace*{\fill}
\begin{center}
\begin{minipage}{0.85\textwidth}
\begin{center}
{\heiti\zihao{4} 摘\quad 要}
\end{center}
\vspace{0.45cm}
\zihao{-4}

本设计基于 STM32F103RCT6（Cortex-M3）微控制器实现了一套桌面用电监控系统。系统不依赖操作系统，以裸机协作方式调度，软件按模块划分：用 PWM 输出测试频率，用定时器输入捕获测量频率，由双通道 DAC 产生正弦波，对三通道 ADC 交流信号采样并计算电参量，再经 TFT LCD 多级菜单显示、通过 USART 二进制帧协议与上位机通信。

硬件方面，系统以 MiniSTM32 开发板为核心，利用 PA8/TIM1 输出 50\% 占空比可调频率方波，PA1/TIM2 输入捕获测量外部信号频率，PA4/PA5 DAC 双通道经 DMA 循环输出 128 点正弦波表，PC0/PC3/PC2 三通道 ADC 在 TIM3 触发下以 6400 Hz 采样率完成电压、电流和漏电电流的定时顺序扫描采样，三路信号共用同一触发节拍，通道间仅有很小的转换时差；对 50 Hz 基波而言，该时差对计算结果影响很小。采集数据经 DMA 搬运后由整型算法库计算有效值、有功功率、视在功率和功率因数。通信协议采用两层设计：传输层以 A5 5A 帧头、CRC-16/Modbus 校验保证可靠性；命令层在帧内承载 ASCII 文本指令，支持 HELP、STATUS?、REPORT ON/OFF、PWM SET、DAC SET 等命令。LCD 显示采用一个主菜单与四个功能页面组成的多页面状态机，串口与 LCD 共用统一监控快照结构，保证两端显示数据来源一致。

经调试验证，系统频率测量结果以 0.01 Hz 为显示分辨率输出，PWM 软件设定范围为 1--100000 Hz，DAC 输出频率 1--1000 Hz 可调。ADC 采样、电参量计算、LCD 显示和串口通信在本次测试条件下运行稳定，能够完成课程设计要求的主要功能。

\sectiontitle{1}{课程设计的目的及要求}

本课程设计以桌面用电监控为主题，基于 STM32F103RCT6 微控制器开发板，设计并实现一套能生成与采集信号、计算电参量、本地显示并远程通信的嵌入式监控系统。设计过程覆盖硬件接口配置、外设驱动、通信协议、算法实现与人机交互，目的是通过完整的系统级实践锻炼嵌入式设计能力。

\phantomsection\label{sec:analysis}
\sectiontitle{2}{系统分析}

本系统的核心任务是对桌面用电环境中的电信号进行生成、采集、计算和展示。系统围绕 STM32F103RCT6 微控制器，综合运用定时器、DAC、ADC、DMA 和 USART 等片上外设，实现信号生成、频率测量、交流采样、电参量计算和串口通信等功能。本章分析系统的功能需求、性能需求与关键算法。

\phantomsection\label{sec:analysis-function}
\subsectiontitle{2.1}{功能分析}

\itemline{1}{频率测试信号生成}

测试频率源由 PA8 的 TIM1\_CH1 通道产生，输出 50\% 占空比方波。默认 50 Hz，可经 LCD 菜单按键或串口命令 PWM SET 在 1--100000 Hz 内调整。定时器工作于向上计数模式，按预分频器（PSC）和自动重装载值（ARR）覆盖宽频率范围，CCR 取 ARR 的一半，得到 50\% 占空比。

\itemline{2}{频率测量}

频率测量利用 PA1 的 TIM2\_CH2 输入捕获完成。TIM2 以 1 MHz 时钟运行（72 MHz / 72），计数器每加 1 对应 1 $\mu$s。相邻上升沿的捕获时间差即为信号周期，再由 $f = 100{,}000{,}000 / T_{\mu s}$ 换算为 Hz$\times$100 格式的整数频率值。16 位计数器的溢出中断把时间高位扩展到 32 位，以支持低频信号测量。

\itemline{3}{正弦波测试信号生成}

正弦波测试信号由 PA4/PA5 的 DAC 双通道生成。系统预先算好 128 点满幅正弦查找表（0--4095，12 位分辨率），运行时按用户设定的幅度和相位缩放后存入 DMA 缓冲区。TIM6 TRGO 以固定采样率触发 DMA 循环搬运，把波形写入 DAC 数据寄存器。支持 SINGLE（单通道正弦 + 单通道中点）与 DUAL（双通道正弦，可设相位差）两种模式，频率 1--1000 Hz、幅度 0--2047 码值可调。

\itemline{4}{桌面用电信号采集与 miniIO 信号处理电路}

系统实际用电信号由 miniIO 副板完成前级检测与处理。miniIO 副板包含交流电压检测、交流电流检测、漏电电流检测、市电频率整形、继电器控制、蜂鸣器控制和 MAX7219 数码管接口等模块。本系统主要使用其中的交流电压、电流、漏电电流和频率检测部分。

交流电压检测链路为：市电电压信号经副板电压检测电路隔离和缩放后，进入运算放大器信号处理电路，并叠加 REF-1.65V 直流偏置，最终输出 VL 信号。VL 为以 1.65 V 为中心的交流采样信号，输入 STM32 的 PC0/ADC1\_IN10 通道。交流电流检测链路为：负载电流经电流互感器转换为小电流信号，经采样电阻和运放处理后输出 iL，输入 STM32 的 PC3/ADC1\_IN13 通道。漏电电流检测链路为：火线和零线共同穿过漏电互感器，当两线电流不平衡时产生差动检测信号，经运放处理后输出 iLK，输入 STM32 的 PC2/ADC1\_IN12 通道。

STM32F103 的 ADC 输入范围为 0--3.3 V，miniIO 副板用 1.65 V 基准偏置把交流信号抬升到该范围内。软件计算电参量时先扣除直流偏置，再求电压、电流和漏电电流的有效值、有功功率、视在功率和功率因数。副板输出的 fr 信号为市电交流频率整形后的方波信号，输入 PA1/TIM2\_CH2，用于定时器输入捕获测量市电频率。

ADC1 配置为扫描模式，由 TIM3 TRGO 以 6400 Hz 触发（50 Hz $\times$ 128 点），每次触发完成 3 通道规则组转换。DMA1\_Channel1 以 2 倍帧长（768 半字）循环搬运，HT 和 TC 中断分别在半帧和整帧完成时进行通道数据拆分，实现连续无丢失采样。DAC 正弦波输出仅作为测试信号源，用于在不接入 miniIO 和实际负载时验证 ADC 采样和电参量计算链路，不属于实际桌面用电检测链路。

\itemline{5}{曲线绘制}

LCD 显示模块在 DA Wave 页面绘制 DAC 双通道波形预览曲线。将 128 点 DAC 样点（0--4095）线性映射到图形区域 y 坐标，采用逐点连线方式绘制。坐标轴含纵轴（幅值方向）、横轴（零线）和刻度标记，CH1 用绿色、CH2 用黄色区分显示。在 AC Measure 页面以文本形式显示电参量数值和 ADC 原始样点范围诊断信息。

\itemline{6}{信号分析}

电参量计算采用整型算法实现，算法库通过 C 接口供主控程序调用。计算内容包括：电压有效值（VRMS）、电流有效值（IRMS）、漏电电流有效值、有功功率（P）、视在功率（S）和功率因数（PF）。所有物理量使用整数表示，避免浮点运算，适合 Cortex-M3 无 FPU 的硬件特点。

\itemline{7}{通讯协议}

系统采用两层协议设计。传输层定义二进制帧结构：A5 5A 帧头 + 版本号 + 帧类型 + 序号 + 长度 + Payload + CRC-16/Modbus 校验，最大 Payload 63 字节。帧类型包括 0x01（命令）、0x81（响应）、0x82（事件/自动上报）、0x83（协议错误）。命令层在帧 Payload 中承载 ASCII 文本指令，支持 HELP、STATUS?、REPORT?、REPORT ON/OFF、PWM SET、DAC SET MODE/FREQ/AMP/PHASE、CAL ZERO 等命令，不区分大小写。

\phantomsection\label{sec:analysis-performance}
\subsectiontitle{2.2}{性能需求分析}

\itemline{1}{硬件电路需求分析。}

STM32F103RCT6 提供 256 KB Flash 和 48 KB RAM，系统时钟 72 MHz。系统不使用堆分配，所有缓冲区均为静态数组。DAC 波形表 128 点$\times$3 组（单通道$\times$2 + 双通道交错）共约 768 字节，ADC DMA 缓冲区 768 半字（1536 字节），协议环形缓冲 256 字节，加上命令响应结构体（4$\times$256 = 1024 字节），总静态内存占用约 4 KB，远低于 48 KB 上限。ADC 时钟需 $\leq$ 14 MHz，实际取 PCLK2/6 = 12 MHz。

\itemline{2}{测量需求分析。}

频率测量分辨率由 TIM2 的 1 MHz 计数时钟决定，周期测量精度为 $\pm$1 $\mu$s。以 50 Hz 信号为例，理论周期 20000 $\mu$s，频率分辨率为 0.01 Hz（对应 $\pm$1 $\mu$s 周期误差引起的频率偏差约 0.003 Hz）。ADC 采样率 6400 Hz，每周期 128 点，满足 Nyquist 定理对 50 Hz 基波及低次谐波的采样要求。RMS 计算基于全周期采样，128 点足以收敛。

\itemline{3}{曲线及参数显示分析。}

LCD 采用 ALIENTEK TFT 屏（16 位并行接口，分辨率 240$\times$320），显示刷新周期 500 ms 或由状态变化即时触发。波形图区域约 216$\times$120 像素，128 点波形经线性插值连线后视觉平滑。电参量以文本格式显示，显示分辨率分别为电压 0.01 V、电流 0.001 A、功率 0.1 W。

\itemline{4}{调试设计分析。}

USART1 配置为 115200 bps、8N1 格式，通过 CH340 USB 转串口芯片连接上位机。协议帧超时保护 300 ms，半帧间隔超时自动重置解析器。环形缓冲区溢出检测保证高负载下不会进入错误状态。自动上报功能可按固定周期（约 1 秒）向串口推送 STATUS 快照，便于实时监控。

\phantomsection\label{sec:analysis-algorithm}
\subsectiontitle{2.3}{算法分析}

本系统各功能模块涉及的核心算法如下：

（1）PWM 频率生成算法。给定目标频率 $f$，先计算总计数值 $N = f_{clk} / f$（$f_{clk} = 72$ MHz），再由 $N$ 拆分出预分频器 $\text{PSC} = \lceil N / 65536 \rceil - 1$ 和自动重装载值 $\text{ARR} = f_{clk} / (f \cdot (\text{PSC}+1)) - 1$。PWM 比较值 CCR = (ARR+1)/2 保证 50\% 占空比。

（2）输入捕获测频算法。TIM2 以 1 MHz 运行，相邻上升沿捕获值相减得到周期 $T$（$\mu$s），频率 $f = 100{,}000{,}000 / T$（单位 Hz$\times$100）。32 位时间戳通过 16 位溢出计数器扩展，支持最长约 71 分钟的无环绕时间表示。

（3）DAC 正弦波生成算法。预计算 128 点满幅正弦查找表 $S[n] = 2048 + 2047 \cdot \sin(2\pi n / 128)$，运行时按幅度参数 $A$ 缩放：$S'[n] = 2048 + (S[n] - 2048) \times A / 2047$。双通道模式下 CH2 引入相位偏移索引 $\Delta = \phi \times 128 / 360$。TIM6 采样率 $f_s = f_{sig} \times 128$，通过调整 PSC 和 ARR 实现。

（4）ADC 采样与数据拆分算法。DMA 以 circular 模式搬运 2 倍帧长（768 半字），每帧包含 3 通道 $\times$ 128 点交错数据。HT 中断处理前半帧、TC 中断处理后半帧，各自将交错序列 V, I, ILK, V, I, ILK, \ldots 拆分为三个独立连续数组。

（5）RMS 有效值算法。对 $N$ 个采样点先减 DC 偏移 $Z$，再计算 $\text{RMS} = \sqrt{\frac{1}{N}\sum_{i=0}^{N-1} (x_i - Z)^2}$。DC 偏移由 CAL ZERO 命令手动校准或自动取算术平均值。中间平方和用 64 位整数累加避免溢出，最终由整数开平方运算求解。

（6）有功功率算法。逐点计算电压、电流去直流后的瞬时功率积 $p_n = (v_n - Z_v) \times (i_n - Z_i)$，取均值后乘组合增益得物理功率。公式为 $P \times 10 = \overline{p} \times G_v \times G_i / 10^{10}$，保留符号（正=消耗，负=回馈）。视在功率 $S = V_{rms} \times I_{rms}$，功率因数 $\text{PF} = |P| / S$，$S = 0$ 时 PF 返回 0。

\textbf{主循环流程：}主循环是系统调度的核心，每轮迭代按以下顺序执行：

（1）调用 app\_capture\_get\_time\_us() 获取统一的 32 位微秒时间戳 now\_us，本轮所有模块使用同一时间基准。

（2）调用 app\_protocol\_task(now\_us) 从环形缓冲区取出字节送入帧解析状态机，解析完成的命令帧交由 app\_command\_handle\_line 处理，返回 display\_changed 标志。

（3）调用 app\_capture\_task() 处理输入捕获超时检测和自动上报节流逻辑。

（4）调用 app\_adc\_task() 检查是否有新的 ADC 帧就绪，若有则复制到工作帧并调用算法库计算电参量。

（6）调用 app\_keys\_poll(now\_us) 扫描按键消抖，再由 app\_ui\_handle\_key 驱动菜单状态机。

（7）调用 app\_monitor\_state\_read 组装最新快照，判断 display\_changed 后调用 app\_display\_task 刷新 LCD。

\textbf{关键设计决策：}

（1）统一监控快照（app\_monitor\_state\_t）。将频率测量、PWM 输出、DAC 配置和 ADC 电参量集中到一个结构体，LCD 显示和串口 STATUS? 命令都从同一份快照读取数据，保证两端显示一致。

（2）ISR 与主循环数据隔离。串口接收采用 256 字节环形缓冲区，ISR 只负责 push、主循环负责 pop；ADC 采用三层快照架构（ISR 双缓冲 $\rightarrow$ 主循环工作帧 $\rightarrow$ 显示帧），临界区极短（仅关中断取帧编号），避免阻塞中断。

（3）Rust/C 混合编程。电参量计算使用 Rust 实现为 no\_std 静态库，通过 cbindgen 自动生成 C 头文件，CMake 构建时自动调用 cargo build 编译。接口边界使用 \texttt{\#[repr(C)]} 和 \texttt{extern "C"} 保证 ABI 兼容。

\phantomsection\label{sec:frequency}
\sectiontitle{4}{频率测量}

频率测量模块包括测试频率信号生成和外部信号频率测量两部分。测试信号由 PA8/TIM1\_CH1 输出 PWM 方波，可通过 LCD 菜单或串口命令调整频率。频率测量由 PA1/TIM2\_CH2 输入捕获实现，通过相邻上升沿时间差计算频率。两部分通过统一的二进制帧协议与上位机通信。

\phantomsection\label{sec:frequency-generate}
\subsectiontitle{4.1}{测试频率生成}

PWM 测试信号由 TIM1 通道 1（PA8）输出，配置为 PWM1 模式、向上计数、50\% 占空比。频率调整的核心是根据目标频率计算定时器的预分频器 PSC 和自动重装载值 ARR。

\textbf{参数计算方法：}定时器时钟 $f_{clk} = 72$ MHz（TIM1 挂载在 APB2 总线上，APB2 预分频为 1，定时器时钟等于系统时钟）。目标总计数值 $N = f_{clk} / f_{target}$。PSC 和 ARR 均为 16 位寄存器（最大值 65535），当 $N > 65535$ 时需要预分频：

$$\text{PSC} = \lceil N / 65536 \rceil - 1$$
$$\text{ARR} = f_{clk} / (f_{target} \times (\text{PSC} + 1)) - 1$$
$$\text{CCR} = (\text{ARR} + 1) / 2$$

以默认 50 Hz 为例：$N = 72{,}000{,}000 / 50 = 1{,}440{,}000$，$\text{PSC} = \lceil 1440000 / 65536 \rceil - 1 = 21$，$\text{ARR} = 72000000 / (50 \times 22) - 1 = 65454$，$\text{CCR} = 32727$。

频率范围限制为 1--100000 Hz，超出范围自动钳位。修改频率时通过 TIM\_PrescalerConfig（立即生效模式）和 TIM\_SetAutoreload 更新参数，并触发 Update 事件使新值立即加载。

\phantomsection\label{sec:frequency-measure}
\subsectiontitle{4.2}{频率测量}

频率测量基于 TIM2 通道 2（PA1）的输入捕获功能。TIM2 配置为 1 MHz 计数时钟（72 MHz / 72），ARR=0xFFFF，以自由运行方式计数，溢出后从 0 重新计数。输入捕获配置为上升沿触发，数字滤波器系数 0x03（4 个采样时钟一致后确认边沿）。

\textbf{测量原理：}每次上升沿触发 CC2 中断，读取 16 位捕获值 CCR2。通过 32 位时间戳扩展函数 make\_capture\_time() 将 16 位值扩展为 32 位微秒时间：高 16 位来自溢出计数器 s\_overflows，低 16 位来自 CCR2。当硬件 Update 标志已置位且捕获值较小（$<$ 32768）时，说明捕获发生在溢出之后，高位需补加 1。

相邻两次捕获的 32 位时间戳相减得到信号周期 $T$（$\mu$s），频率计算为：

$$f \times 100 = 100{,}000{,}000 / T$$

结果以 Hz$\times$100 格式存储，分辨率 0.01 Hz。

\textbf{超时处理：}主循环中 app\_capture\_task() 检测距离上次捕获超过 2 秒无边沿时，将频率清零并清除捕获标志，防止 LCD 和串口显示过期的频率值。

\textbf{上报节流：}自动上报采用双重策略——周期上报每 1 秒触发一次；频率变化时额外上报，但两次变化上报间隔不小于 200 ms 以防抖。

\phantomsection\label{sec:adc}
\sectiontitle{5}{交流ADC采样}

交流采样模块负责对桌面交流电信号进行定时 ADC 转换并通过 DMA 传送至内存。系统同时具备 DAC 正弦波信号生成能力，可作为测试信号源供采样验证。本章包括测试信号生成、信号采集、通信协议与波形绘制。

\phantomsection\label{sec:adc-generate}
\subsectiontitle{5.1}{测试电压电流生成与实际信号来源}

本系统存在两类交流信号来源。第一类为 DAC 生成的可控正弦波测试信号，用于在不接入 miniIO 和实际负载的情况下验证 ADC 采样、DMA 搬运、电参量计算和 LCD/串口显示链路。第二类为 miniIO 副板输出的实际桌面用电检测信号，包括 VL、iL 和 iLK 三路模拟量，以及 fr 频率方波。DAC 测试信号主要用于算法与采样链路调试，miniIO 输出信号用于系统联机后的实际用电监控。

测试电压和电流信号由 PA4/PA5 的 DAC 双通道生成，采用查找表 + DMA 循环 + 定时器触发的方案。

\textbf{正弦查找表：}预计算 128 点满幅正弦表 s\_sine\_fullscale[128]，值域 0--4095（12 位 DAC 全幅），以 2048 为零点上下对称。该表在编译期确定，运行时不做 sin() 浮点运算。

\textbf{幅度缩放：}对每个满幅样点做线性缩放，公式为：

$$S'[n] = 2048 + (S[n] - 2048) \times A / 2047$$

其中 $A$ 为用户设定的幅度码值（0--2047），结果钳位到 0--4095。

\textbf{相位偏移：}DUAL 模式下 CH2 相对 CH1 的相位偏移通过索引偏移实现：$\Delta = \phi \times 128 / 360$，CH2 读取查找表时索引为 $(n + \Delta) \bmod 128$。

\textbf{DMA 传输：}SINGLE 模式使用 DMA2\_Channel3 向 DHR12R1 搬运 16 位数据（CH1 正弦波），DMA2\_Channel4 向 DHR12R2 搬运 16 位数据（CH2 固定中点 2048）。DUAL 模式仅用 DMA2\_Channel3 向 DHR12RD 搬运 32 位数据，一次搬运同时更新两通道。DMA 配置为 circular 模式，自动循环读取 128 点缓冲区。

\textbf{触发时序：}TIM6 以 TRGO（Update 事件）触发 DAC 转换。采样率 $f_s = f_{sig} \times 128$，由 TIM6 的 PSC 和 ARR 决定。以 50 Hz 信号为例，$f_s = 6400$ Hz，$N = 72{,}000{,}000 / 6400 = 11250$，PSC = 0，ARR = 11249。

\phantomsection\label{sec:adc-sampling}
\subsectiontitle{5.2}{信号采集}

\textbf{ADC 输入信号来源：}在实际桌面用电监控场景下，ADC 的三路输入信号均来自 miniIO 副板的调理输出。市电电压经副板电压互感器隔离缩放后，由运放叠加 1.65 V 直流偏置输出 VL（0--3.3 V 范围内以 1.65 V 为中心的交流信号），送入 PC0/ADC1\_IN10。负载电流经副板电流互感器转换为小信号，由采样电阻和运放调理后输出 iL，送入 PC3/ADC1\_IN13。漏电电流由火线和零线共同穿过漏电互感器产生差动信号，经运放调理后输出 iLK，送入 PC2/ADC1\_IN12。市电频率由副板整形为低压方波 fr，送入 PA1/TIM2\_CH2 进行输入捕获。在调试阶段，也可将 DAC 输出通过跳线连接至 ADC 输入端进行环回测试，此时 ADC 采集的是 STM32 自身生成的正弦波信号。

ADC 采样硬件链路：TIM3（6400 Hz TRGO）$\rightarrow$ ADC1（扫描模式，3 通道）$\rightarrow$ DMA1\_Channel1（循环搬运）。

\textbf{采样配置：}ADC1 配置为独立模式、非连续扫描、TIM3 TRGO 外部触发。规则组 3 个 Rank 分别为 PC0/IN10（电压 VL）、PC3/IN13（电流 iL）、PC2/IN12（漏电电流 iLK），采样时间均取 239.5 ADC 时钟周期（约 20 $\mu$s @ 12 MHz），较长的采样窗口有利于降低外部信号处理电路输出阻抗的影响。

\textbf{DMA 传输：}DMA1\_Channel1 配置为外设到内存、半字宽度、循环模式，缓冲区大小为 $2 \times 3 \times 128 = 768$ 半字。HT（半传输）中断在前半帧 [0, 384) 写完时触发，TC（传输完成）中断在后半帧 [384, 768) 写完时触发。

\textbf{三层快照架构：}为解决 DMA ISR、主循环和 LCD/串口之间的数据竞争，采用三层缓冲设计：

第一层（ISR 写缓冲）：DMA HT/TC 中断中，deinterleave\_frame() 从 DMA 缓冲区将交错数据分离到双缓冲 s\_adc\_frame[2] 的写侧，完成后切换 write/ready 索引。

第二层（主循环工作帧）：app\_adc\_task() 在极短临界区（仅关中断取 ready 帧编号并清标志）后，将整帧数据从 ISR 缓冲复制到工作帧 s\_adc\_work\_frame（临界区外，不阻塞中断），再调用算法库计算电参量。

第三层（显示帧快照）：计算完成后将工作帧复制到 s\_adc\_display\_frame，LCD 波形绘制、串口 STATUS 输出和 CAL ZERO 校准统一读取此快照，保证数据一致性。

\textbf{DAC 参数修改流程：}收到 DAC SET 命令后，先读取当前配置（app\_dac\_get\_config），修改对应字段，调用 app\_dac\_apply\_config 执行：停止 DMA 和 TIM6 $\rightarrow$ 重建波形缓冲区 $\rightarrow$ 重新配置 DMA $\rightarrow$ 更新 TIM6 参数 $\rightarrow$ 重新启动。再读回配置（因钳位可能改变实际值）后格式化响应。

\textbf{CAL ZERO 校准：}取显示帧各通道 128 点算术平均值作为 DC 偏移补偿值。CAL ZERO 建议在无交流输入或输入稳定为静态中点时执行。若在带有交流信号、非整周期信号或偏置异常状态下执行，可能引入零点误差，导致后续 VRMS、IRMS、P 和 PF 计算产生偏差。

\phantomsection\label{sec:adc-waveform}
\subsectiontitle{5.4}{波形曲线绘制}

波形绘制在 LCD 的 DA Wave 页面完成，显示 DAC 双通道的 128 点波形预览。

\textbf{坐标映射：}12 位 DAC 样点（0--4095）映射到图形区域 y 坐标：

$$y = \text{top} + (4095 - \text{sample}) \times (\text{height} - 1) / 4095$$

其中图形区域 top = 132、height = 120 像素。样点值越大（电压越高），y 坐标越小（屏幕上方），符合直觉。

\textbf{绘制方法：}采用逐点连线方式。128 个样点均匀分布在图形宽度内，x 坐标为 $\text{left} + i \times (\text{width} - 1) / 127$。相邻两点之间调用 LCD\_DrawLine 绘制线段，形成连续波形曲线。CH1 使用绿色、CH2 使用黄色。

\textbf{坐标轴：}绘制纵轴（左侧灰线，含箭头）、横轴（零线，中线位置，含箭头）、1/4 和 3/4 处幅值刻度线、横轴等距时间刻度线。

\textbf{刷新策略：}LCD 刷新由两种条件触发——状态变化时即时刷新（display\_changed 标志）或固定 500 ms 周期刷新。换页时先清除内容区域避免旧页面残留。

\phantomsection\label{sec:calculation}
\sectiontitle{6}{电参量计算与分析}

电参量计算是系统的核心功能之一，将 ADC 采集的原始样点转换为有意义的电气物理量。本章介绍交流电参量计算方法、本系统的程序实现方案以及与上位机的通信设计。

\phantomsection\label{sec:calculation-method}
\subsectiontitle{6.1}{交流电参量计算方法}

交流电参量的计算方法主要有以下三类：

\textbf{（1）直接计算方法。}对采样得到的离散时域信号直接进行数学运算。电压有效值（RMS）定义为 $V_{RMS} = \sqrt{\frac{1}{N}\sum_{i=0}^{N-1}(V_i - V_{dc})^2}$，有功功率定义为 $P = \frac{1}{N}\sum_{i=0}^{N-1} V_i \times I_i$。该方法计算简单、实时性好，对采样率要求不高，适合嵌入式系统。

\textbf{（2）DFT计算方法。}将时域信号变换到频域，提取基波和各次谐波分量。可分别计算各谐波的幅值和相位，进而得到各次谐波功率。计算量为 $O(N^2)$，在样点数较多时开销较大。

\textbf{（3）FFT计算方法。}DFT 的快速算法，计算量 $O(N \log N)$。适合样点数较多（通常要求 2 的幂次）且需要频谱分析的场景。在 Cortex-M3 上实现 FFT 需要额外的整型数蝶形运算库。

系统选择直接计算方法：每周期 128 点采样足以满足基波 RMS 与功率精度，计算量小、实时性好，且无需复数运算与频谱分析；Cortex-M3 无硬件 FPU，整数运算效率更高。

\phantomsection\label{sec:calculation-program}
\subsectiontitle{6.2}{程序实现}

电参量计算由 pm\_calc\_electrical() 函数完成，以 no\_std 整型方式实现，通过 C 接口供 C 代码调用。整数运算无运行时浮点开销，契合 Cortex-M3 无 FPU 的硬件特点。计算流程如下：

\textbf{（1）DC 偏移确定。}各通道独立计算直流分量。若已通过 CAL ZERO 命令手动校准（provided\_zero $\neq$ 0），直接使用校准值；否则从本批 128 个样点取算术平均值 $Z = \frac{1}{N}\sum_{i=0}^{N-1} x_i（N=128）$ 作为自动估算的 DC 偏移。

\textbf{（2）RMS 码值计算。}对每通道去直流后计算均方根值：

$$\text{RMS}_{code} = \sqrt{\frac{1}{N}\sum_{i=0}^{N-1}(x_i - Z)^2}$$

中间平方和使用 u64（64 位无符号整数）累加，12 位 ADC 码值的平方最大 $2047^2 \approx 4.19 \times 10^6$，128 点累加最大约 $5.37 \times 10^8$，远小于 u64 上限 $1.84 \times 10^{19}$，不会溢出。开平方运算采用高效的整数开平方算法实现，返回所求平方根的整数部分。

\textbf{（3）物理量换算。}RMS 码值通过校准增益转换为物理量：

$$V_{rms} \times 100 = \text{RMS}_{code} \times G_v / 1000$$
$$I_{rms} \times 1000 = \text{RMS}_{code} \times G_i / 1000$$

其中 $G_v$、$G_i$ 为校准增益$\times$1000（默认 1000，即 1:1）。

\textbf{（4）有功功率计算。}逐点计算去直流后的瞬时功率积并取均值：

$$P_{code} = \frac{1}{N}\sum_{i=0}^{N-1}(v_i - Z_v) \times (i_i - Z_i)$$

有功功率物理量：

$$P \times 10 = P_{code} \times G_v \times G_i / 10^{10}$$

除数为 $10^{10}$ 的原因是 $V_{rms}\times100$（单位 0.01 V）和 $I_{rms}\times1000$（单位 0.001 A）的单位缩放因子不同，$\times$增益$^2$后需除以增益$^2 \times 10000$。结果保留符号，正值表示消耗功率，负值表示回馈功率。

\textbf{（5）视在功率：}$S \times 10 = V_{rms\times100} \times I_{rms\times1000} / 10000$。

\textbf{（6）功率因数：}$\text{PF} \times 1000 = |P| \times 1000 / S$，钳位到 [0, 1000] 区间。$S = 0$ 时（无负载）PF 返回 0，因为无负载时功率因数无物理意义。

\subsectiontitle{7.3}{AD采样调试}

ADC 采样调试分为零输入基线测试、DAC 环回测试和波形显示测试。零输入测试时不接外部交流信号，观察 PC0、PC3、PC2 三个 ADC 通道的原始码值是否稳定在中点附近。DAC 环回测试时，将 PA4/DAC\_OUT1 和 PA5/DAC\_OUT2 通过跳线接入 ADC 输入端，设置不同幅值、频率和相位，观察采样结果和电参量变化。

\textbf{（1）零信号基线测试。}不接入外部信号，发送 STATUS? 查看各通道 ADC 原始码值范围。PC0 和 PC3 在无信号时应稳定在中点附近（约 2048），若某通道码值明显偏大或偏小，说明引脚被外部电路拉偏。

\phantomsection\label{sec:summary}
\sectiontitle{8}{总结与展望}

本次课程设计完成了基于 STM32F103RCT6 的桌面用电监控系统：信号生成、频率测量、DAC 输出、ADC 采样、电参量计算、LCD 显示与串口通信各模块均已实现并联调通过。系统采用裸机协作调度，无 RTOS 依赖，软件分层清晰，LCD 与串口通过统一监控快照保持数据一致。

\textbf{调试过程中遇到的主要问题及解决办法：}

（1）PC1（ADC1\_IN11）电流通道读数异常。现象：DAC DUAL 环回测试时 IRMS 始终约为 0.056，远小于预期。排查发现 MiniSTM32 开发板的 PC1 与板载触摸屏 T\_PEN 共享引脚，硬件上拉电阻将 PC1 拉至约 3V，导致 ADC 读数严重偏高。解决办法：将电流采样通道迁移至 PC3（ADC1\_IN13），避开触摸屏干扰。

（2）ADC 采样数据竞争。现象：LCD 显示的电参量偶尔跳变，串口 STATUS 返回的 ADC 样点范围不一致。根因是 DMA ISR 正在写入缓冲区时主循环同时在读取。解决办法：设计三层快照架构——ISR 写双缓冲、主循环极短临界区取帧号、计算后复制到显示帧——从根本上消除数据竞争。

（3）命令响应栈溢出。现象：增大 RESPONSE\_MAX（响应行数）后设备硬故障无响应。根因是 app\_command\_result\_t 结构体在 dispatch\_command\_frame() 栈上分配，4$\times$256 = 1024 字节已接近栈上限。解决办法：将 RESPONSE\_MAX 限制为 4，并将 HELP 命令的两条 DAC 帮助合并为一行以适配行数限制。

（4）CAL ZERO 执行时机不当导致电参量异常。现象：在 DAC 输出交流信号时执行 CAL ZERO 后，VRMS/IRMS/P 等参数出现明显偏差。根因是校准函数在带有交流信号的状态下记录了非静态零点，引入零点误差，导致后续 RMS 和功率计算偏差。解决办法：在文档和帮助信息中明确 CAL ZERO 应在无交流输入或输入稳定为静态中点时执行。

\textbf{心得体会：}

本次课程设计中最具挑战性的部分是数据竞争问题的排查——DMA 中断与主循环并发访问同一缓冲区的问题在代码审查中难以发现，需要结合现象反复推理数据时序。三层快照架构的设计过程加深了我对嵌入式并发数据一致性的理解。

在团队协作方面，与同学在引脚分配、协议格式和算法选择上进行了充分讨论，把常量集中到 app\_config.h、把帧定义放进共享的 app\_protocol\_defs.h，使固件与上位机工具的协议保持一致。AI 工具在代码审查、Bug 分析和架构建议方面提供了有价值的辅助，尤其是在数据竞争根因分析和栈溢出问题定位中发挥了关键作用；但 AI 生成的代码仍需人工审查边界条件和嵌入式约束。

\textbf{改进方向：}

（1）增加 FFT 频谱分析功能，支持谐波检测和 THD 计算。

（2）将串口协议升级为支持批量数据传输，便于绘制实时波形。

（3）增加 SD 卡数据记录功能，支持长时间监测数据离线分析。

（4）优化 LCD 显示刷新策略，采用局部刷新减少闪烁。
